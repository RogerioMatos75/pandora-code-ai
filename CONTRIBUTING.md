# Guia de Contribuição

## 🎯 Áreas Prioritárias para Feedback

### 1. Integração com DeepSeek
- Performance do modelo local
- Qualidade das sugestões
- Tempo de resposta

### 2. UX/UI
- Clareza das mensagens
- Facilidade de uso
- Visual dos resultados

### 3. Código
- Estrutura do projeto
- Padrões utilizados
- Cobertura de testes

## 📝 Processo de Review

1. **Código**
   - Seguir padrões TypeScript
   - Manter cobertura de testes
   - Documentar alterações

2. **Documentação**
   - Atualizar README se necessário
   - Documentar novas features
   - Adicionar exemplos de uso

3. **Testes**
   - Adicionar testes unitários
   - Verificar edge cases
   - Manter mocks atualizados

## 🔍 Checklist para PRs

- [ ] Testes passando
- [ ] Lint sem erros
- [ ] Documentação atualizada
- [ ] Changelog atualizado
